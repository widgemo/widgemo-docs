# Header & Footer Zone Config

Widgemo has three named zones: `header`, `content`, and `footer` â€” all under `config.zones`. This page covers the **header and footer zones**, which are optional and share the `ZoneConfig` shape. The content zone is required and uses a separate, richer type â€” [Content Config](/core/reference/content-config) â€” because it owns the entire data rendering pipeline: mode, fields, actions, sorting, filtering, pagination, and status.

## Shape

| Field | Type | Default | Description |
|---|---|---|---|
| `title` | `string \| ReactNode \| (data, id?) => string` | â€” | Zone title. Pass a function to derive it from live data; receives the full data array and the optional instance `id`. |
| `subtitle` | `string \| ReactNode \| (data, id?) => string` | â€” | Zone subtitle. Same function signature as `title`. |
| `icon` | `string \| IconConfig \| { src, size?, color? }` | â€” | Icon displayed beside (or above) the title. |
| `layout` | `ZoneLayout` | â€” | Positional layout for the zone header area. |
| `actions` | `ActionConfig[]` | â€” | Zone-level action buttons. |
| `actionOverflow` | `{ maxInline?, menuLabel?, menuTooltip?, indicator? }` | â€” | Action overflow behaviour. See [Action Config](/core/reference/action-config#action-overflow). |
| `enabled` | `boolean` | `true` | Show or hide the zone entirely. |
| `theme` | `string` | â€” | Registered theme name applied to this zone only. |
| `themeOverrides` | `Partial<ZoneTheme>` | â€” | Inline theme property overrides. See [Theme API](/core/reference/theme-api). |
| `style` | `CSSProperties` | â€” | Inline CSS on the zone element. |
| `className` | `string` | â€” | Additional CSS class on the zone element. |
| `customSections` | `ReactNode[]` | â€” | Extra React nodes rendered inside the zone chrome. |

## Dynamic title and subtitle

`title` and `subtitle` both accept a function `(data) => string`. The function receives the live data array on every render â€” use it to reflect counts, filters, or any derived summary without external state.

<WidgemoShowcase
  description="title and subtitle as functions: title counts all rows, subtitle counts active-only."
  data={teamData}
  config={{
    zones: {
      header: {
        title:    (data) => `Team (${data.length} members)`,
        subtitle: (data) => `Active: ${data.filter(d => d.status === 'active').length}`,
        actions: [{ id: 'export', label: 'Export', icon: 'export', placement: 'pinned' }],
      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name',   label: 'Name' },
            { key: 'status', label: 'Status', renderAs: 'badge' },
          ],
          layout: { type: 'auto' },
        },
      },
    },
  }}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
header: {
  title:    (data) => `Team (${data.length} members)`,
  subtitle: (data) => `Active: ${data.filter(d => d.status === 'active').length}`,
},
```

Both `title` and `subtitle` accept the same function signature: `(data: Entity[], id?: string) => string`. The `id` argument is the value of the `id` **prop** on `<Widgemo>` (i.e. the HTML element id) â€” useful when you share a single function reference across multiple instances:

```tsx
const sharedTitle = (data, id) => `[${id ?? 'widget'}] ${data.length} items`;

widgets.map((w) => (
  <Widgemo
    key={w.id}
    id={w.id}
    data={w.rows}
    config={{
      zones: { header: { title: sharedTitle }, content: { /* â€¦ */ } },
    }}
  />
));\n```

`icon` accepts a registered icon name (string), an `IconConfig` object, or a `{ src, size?, color? }` object for image/URL icons.

## Collapse

Collapse is a **top-level `WidgemoConfig` field** â€” not a `ZoneConfig` field. It collapses the entire content and footer area; the toggle button always renders inside the header zone.

<WidgemoShowcase
  description="Widget starts collapsed. The toggle in the header expands and collapses the content and footer."
  data={teamData}
  config={{
    collapse: { initialState: 'collapsed', button: true },
    zones: {
      header: {
        title: 'Team Directory',
        subtitle: 'Click the toggle to expand',
      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name', label: 'Name' },
            { key: 'role', label: 'Role' },
          ],
          layout: { type: 'auto' },
        },
      },
      footer: {
        subtitle: 'Last synced just now',
      },
    },
  }}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
// collapse sits at the top level of WidgemoConfig â€” not inside a zone
const config = {
  collapse: { initialState: 'collapsed', button: true },
  zones: {
    header: { title: 'Team Directory', subtitle: 'Click to expand' },
    content: { /* â€¦ */ },
    footer: { subtitle: 'Last synced just now' },
  },
};
```

| `initialState` | Effect |
|---|---|
| `'expanded'` | Widget starts open. Toggle shown if `button: true`. Default. |
| `'collapsed'` | Widget starts with content and footer hidden. Toggle shown if `button: true`. |
| `'fixed'` | Widget cannot be toggled. `button` defaults to `false`. |

The toggle button appears in the **header zone** â€” a header zone must be present for it to render. The footer is also hidden when collapsed.

## `ZoneLayout`

`layout` controls how the zone header area is composed â€” the relative positions of icon, title, subtitle, and actions.

### `ZoneLayout` options

| Option | Values | Default | Effect |
|---|---|---|---|
| `orientation` | `'horizontal'` \| `'vertical'` | `'horizontal'` | Stack zone sections side-by-side or top-to-bottom. |
| `iconPosition` | `'left'` \| `'above'` | `'left'` | Icon appears to the left of the title block, or centred above it. |
| `actionsPosition` | `'right'` \| `'below'` | `'right'` | Action buttons appear at the trailing edge, or below the title block. |
| `titlePosition` | `'left'` \| `'center'` \| `'right'` | `'left'` | Horizontal alignment of the title+subtitle block within the zone. |
| `subtitlePosition` | `'below'` \| `'inline'` | `'below'` | Subtitle stacks under the title, or renders as a separate inline column. |

## `themeOverrides`

`themeOverrides` accepts a partial `ZoneTheme` to customise just this zone's colours, fonts, spacing, or background without affecting other zones or the global theme.

```tsx
header: {
  title: 'Team Directory',
  themeOverrides: {
    background: 'linear-gradient(135deg, #4f46e5 0%, #7c3aed 100%)',
    titleColor: '#ffffff',
    subtitleColor: 'rgba(255,255,255,0.75)',
  },
},
```

For the full `ZoneTheme` shape and theme registration, see [Theme API](/core/reference/theme-api).
