# Header & Footer Zone Config

Widgemo has three named zones: `header`, `content`, and `footer` — all under `config.zones`. This page covers the **header and footer zones**, which are optional and share the `ZoneConfig` shape. The content zone is required and uses a separate, richer type — [Content Config](/core/reference/content-config) — because it owns the entire data rendering pipeline: mode, fields, actions, sorting, filtering, pagination, and status.

## Shape

| Field | Type | Default | Description |
|---|---|---|---|
| `title` | `string \| ReactNode \| (data) => string` | — | Zone title. Pass a function to derive it from live data. |
| `subtitle` | `string \| ReactNode \| (data, id?) => string` | — | Zone subtitle. Pass a function to derive it from live data. |
| `titleRenderer` | `(data, id?) => string` | — | Alternative dynamic title. Receives the instance `id` as a second arg — useful when multiple Widgemos share one renderer function. Takes precedence over `title` when both are set. |
| `subtitleRenderer` | `(data, id?) => string` | — | Alternative dynamic subtitle. Same `id` use-case as `titleRenderer`. Takes precedence over `subtitle` when both are set. |
| `icon` | `string \| IconConfig \| { src, size?, color? }` | — | Icon displayed beside (or above) the title. |
| `layout` | `ZoneLayout` | — | Positional layout for the zone header area. |
| `actions` | `ActionConfig[]` | — | Zone-level action buttons. |
| `actionOverflow` | `{ maxInline?, menuLabel?, menuTooltip?, indicator? }` | — | Action overflow behaviour. See [Action Config](/core/reference/action-config#action-overflow). |
| `collapse` | `{ initialState, button? }` | — | Collapse behaviour for this zone. |
| `enabled` | `boolean` | `true` | Show or hide the zone entirely. |
| `theme` | `string` | — | Registered theme name applied to this zone only. |
| `themeOverrides` | `Partial<ZoneTheme>` | — | Inline theme property overrides. See [Theme API](/core/reference/theme-api). |
| `style` | `CSSProperties` | — | Inline CSS on the zone element. |
| `className` | `string` | — | Additional CSS class on the zone element. |
| `customSections` | `ReactNode[]` | — | Extra React nodes rendered inside the zone chrome. |

## Dynamic title and subtitle

`title` and `subtitle` both accept a function `(data) => string`. The function receives the live data array on every render — use it to reflect counts, filters, or any derived summary without external state.

<WidgemoShowcase
  description="title and subtitle as functions: title counts all rows, subtitle counts active-only."
  data={teamData}
  config={{
    zones: {
      header: {
        title:    (data) => `Team (${data.length} members)`,
        subtitle: (data) => `Active: ${data.filter(d => d.status === 'active').length}`,
        actions: [{ id: 'export', label: 'Export', icon: 'export', placement: 'pinned' }],
      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name',   label: 'Name' },
            { key: 'status', label: 'Status', renderAs: 'badge' },
          ],
          layout: { type: 'auto' },
        },
      },
    },
  }}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
header: {
  title:    (data) => `Team (${data.length} members)`,
  subtitle: (data) => `Active: ${data.filter(d => d.status === 'active').length}`,
},
```

**`titleRenderer` and `subtitleRenderer`** exist for one specific case: when you need the Widgemo instance `id` to differentiate multiple Widgemos sharing the same renderer function — for example, rendering several Widgemos in a loop with one shared header function. Both receive `(data, id?)` and take precedence over `title`/`subtitle` when set. For all other dynamic title needs, the function form of `title` / `subtitle` is sufficient.

```tsx
// Only use titleRenderer when you need the instance id
const sharedTitle = (data, id) => `${id ?? 'Widget'}: ${data.length} items`;

widgets.map((w) => (
  <Widgemo
    key={w.id}
    data={w.rows}
    config={{
      id: w.id,
      zones: { header: { titleRenderer: sharedTitle }, content: { /* … */ } },
    }}
  />
));
```

`icon` accepts a registered icon name (string), an `IconConfig` object, or a `{ src, size?, color? }` object for image/URL icons.

      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name',   label: 'Name' },
            { key: 'role',   label: 'Role' },
            { key: 'status', label: 'Status', renderAs: 'badge' },
          ],

        },
      },
    },
  }}

```tsx

// Image/URL icon

```

## Collapse

`collapse.initialState` controls whether the zone starts expanded or collapsed. `collapse.button` shows or hides the toggle.

<WidgemoShowcase
  description="Footer starts collapsed. Click the toggle to expand it."
  data={teamData}
  config={{
    zones: {
      header: {
        title: 'Team Directory',
        subtitle: 'Click the footer toggle below',
      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name', label: 'Name' },
            { key: 'role', label: 'Role' },
          ],
          layout: { type: 'auto' },
        },
      },
      footer: {
        subtitle: 'Last synced just now',
        collapse: { initialState: 'collapsed', button: true },
      },
    },
  }}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
footer: {
  subtitle: 'Last synced just now',
  collapse: { initialState: 'collapsed', button: true },
},
```

| `initialState` | Effect |
|---|---|
| `'expanded'` | Zone starts open. Toggle is shown if `button: true`. Default. |
| `'collapsed'` | Zone starts closed. Toggle is shown if `button: true`. |
| `'fixed'` | Zone cannot be toggled. `button` defaults to `false`. |

## `ZoneLayout`

`layout` controls how the zone header area is composed — the relative positions of icon, title, subtitle, and actions.

### `ZoneLayout` options

| Option | Values | Default | Effect |
|---|---|---|---|
| `orientation` | `'horizontal'` \| `'vertical'` | `'horizontal'` | Stack zone sections side-by-side or top-to-bottom. |
| `iconPosition` | `'left'` \| `'above'` | `'left'` | Icon appears to the left of the title block, or centred above it. |
| `actionsPosition` | `'right'` \| `'below'` | `'right'` | Action buttons appear at the trailing edge, or below the title block. |
| `titlePosition` | `'left'` \| `'center'` \| `'right'` | `'left'` | Horizontal alignment of the title+subtitle block within the zone. |
| `subtitlePosition` | `'below'` \| `'inline'` | `'below'` | Subtitle stacks under the title, or renders as a separate inline column. |

## `themeOverrides`

`themeOverrides` accepts a partial `ZoneTheme` to customise just this zone's colours, fonts, spacing, or background without affecting other zones or the global theme.

```tsx
header: {
  title: 'Team Directory',
  themeOverrides: {
    background: 'linear-gradient(135deg, #4f46e5 0%, #7c3aed 100%)',
    titleColor: '#ffffff',
    subtitleColor: 'rgba(255,255,255,0.75)',
  },
},
```

For the full `ZoneTheme` shape and theme registration, see [Theme API](/core/reference/theme-api).
