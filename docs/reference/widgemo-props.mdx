# Widgemo Props

`WidgemoProps` is the runtime prop contract for the main `<Widgemo>` component.

For host-level table CSS interaction guidance, see [Host CSS and Table Layout](/core/reference/host-css-table-layout).

<WidgemoShowcase

  description="A minimal Widgemo instance rendered directly inside the docs using the same runtime prop surface described below."
  data={[
    { name: 'Aurora Chen', role: 'Product Designer', status: 'active' },
    { name: 'Mateo Silva', role: 'Engineer', status: 'pending' },
    { name: 'Priya Nair', role: 'Operations', status: 'inactive' },

        },
      },
    },
  }}

## Props

| Prop | Type | Required | Description |
|---|---|---|---|
| `data` | `Entity[]` | Yes | Input dataset for rendering |
| `config` | `WidgemoConfig` | No | Layout and behaviour configuration |
| `className` | `string` | No | Additional CSS class on the root element |
| `id` | `string` | No | HTML `id` on the root element |
| `loading` | `boolean` | No | External loading state (overrides `config.zones.content.status`) |
| `error` | `any` | No | External error state |
| `onRetry` | `() => void` | No | Retry callback when in error state |

## `onRetry` Behavior and Precedence

- Top-level `onRetry` supports externally managed error flow (`error` prop) and runtime boundary retry (fallback `WidgemoRuntimeBoundary` retry button).
- Content-level retry is configured in `config.zones.content.errorState.retry`.
- Precedence in external error flow: when `error` is provided and top-level `onRetry` is set, Widgemo wires that top-level callback into the content error retry action. If `errorState.retry` is an object, its existing fields such as `label` are retained while `errorState.retry.onRetry` is overridden.
- Current external-flow nuance: if `config.zones.content.errorState.retry` is `false`, retry remains disabled even when top-level `onRetry` is provided with `error`.

## `widgemoRegistry` Methods

| Method | Signature | Description |
|---|---|---|
| `registerWidgemoRenderAs` | `(entry: RenderAsRegistryEntry) => void` | Register a custom field renderer |
| `getWidgemoRenderAs` | `(name: string) => RenderAsRegistryEntry \| null` | Look up a renderer |
| `renderWidgemoRenderAs` | `(value, renderAs, options, entity) => ReactNode` | Render via registry |
| `registerWidgemoMode` | `(entry: ModeRegistryEntry) => void` | Register a custom view mode |
| `getWidgemoMode` | `(name: string) => ModeRegistryEntry \| null` | Look up a mode |
| `getModeComponent` | `(name: string) => ComponentType \| null` | Get mode component |
| `getRegisteredModes` | `() => string[]` | List all registered mode names |
| `registerWidgemoHook` | `(entry: HookRegistryEntry) => void` | Register a lifecycle hook |
| `getWidgemoHook` | `(name: string) => HookRegistryEntry \| null` | Look up a hook |
| `executeWidgemoHook` | `(name: string, ...args) => unknown` | Execute a hook by name |
| `registerWidgemoIcon` | `(entry: IconRegistryEntry) => void` | Register a custom icon |
| `getWidgemoIcon` | `(name: string) => IconRegistryEntry \| null` | Look up an icon |
| `renderWidgemoIcon` | `(props: { name, size?, className?, color? }) => ReactNode` | Render icon via registry |
| `registerWidgemoFieldType` | `(entry: FieldTypeRegistryEntry) => void` | Register a custom field type |
| `getWidgemoFieldType` | `(name: string) => FieldTypeRegistryEntry \| null` | Look up a field type |
| `renderWidgemoField` | `(value, config, entity) => ReactNode` | Render field via registry |

## `InteractionContext` Shape

```ts
interface InteractionContext {
  kind:             InteractionKind;  // 'zone-action' | 'item-action' | 'item-click' | 'item-drop' | …
  interactionId:    string;
  interactionLabel: string;
  entity?:          Entity;           // set for item-level interactions
  data:             Entity[];         // full data array in the active render scope
  zone:             string;
  nativeEvent?:     React.MouseEvent | React.DragEvent;
  from?:            InteractionBoardLocation;  // board drag source
  to?:              InteractionBoardLocation;  // board drag destination
}
```
