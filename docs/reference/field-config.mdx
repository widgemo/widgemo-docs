# Field Config

`FieldConfig` defines how each field is rendered. Most options are cross-mode, while some are mode-specific, such as `sortable`, `width`, and `align` in table mode and `span` in grid layouts.

For host-level table CSS interaction guidance, see [Host CSS and Table Layout](/core/reference/host-css-table-layout).

## FieldConfig Interface

```ts
interface FieldConfig {
  key: string;                    // entity property key (required)
  label?: string;                 // display label
  type?: 'text' | 'number' | 'date' | 'time' | 'datetime' | 'timestamp' | 'duration' | 'boolean' | 'select' | 'relation' | 'textarea' | 'email' | 'url' | 'image' | 'swatch';
  sortable?: boolean;
  width?: number | string;        // table column width
  align?: 'left' | 'center' | 'right';
  showLabel?: boolean;            // show label next to value in card views
  wrap?: boolean;                 // true = wrap, false = truncate + tooltip (default)
  visible?: boolean;              // false = hide entirely
  condition?: (entity) => boolean; // per-row hide condition
  formatter?: (value, entity) => unknown; // transform raw value before rendering
  span?: number;                  // grid column span
  booleanTrueLabel?: string;
  booleanFalseLabel?: string;
  imageOptions?: { circular?, width?, height?, objectFit?, border?, shadow?,
    backgroundColor?, borderRadius?, lightbox?, lazy?, alt? };
  renderAs?: string;
  renderAsOptions?: RenderAsOptions;
}
```

### Implementation Notes

- `renderAs` takes precedence over `type`.
- Temporal contract: `date`, `time`, `datetime`, and `timestamp` require `renderAsOptions.parseMode`; `duration` requires `renderAsOptions.unit`.
- `booleanTrueLabel` and `booleanFalseLabel` are applied only for `type: 'boolean'` fallback rendering.
- `formatter` runs before type rendering. For domain-specific boolean wording or logic, prefer a formatter plus `renderAs` or `type: 'text'` instead of relying on boolean labels.
- `visible` is currently enforced in table mode field filtering.
- `span` applies to item layouts that use CSS grid field placement.

## Built-in `renderAs` Options

### `badge`

```tsx
{
  key: 'status', label: 'Status', renderAs: 'badge',
  renderAsOptions: {
    colorMap: {
      active:   { background: '#198754', text: '#fff' },
      pending:  { background: '#ffc107', text: '#333' },
      inactive: { background: '#6c757d', text: '#fff', icon: 'close', iconPosition: 'left' },
    },
    style: 'badge',  // 'badge' (default) | 'inline'
    size: 'md',      // 'sm' | 'md' (default) | 'lg'
    className: 'status-badge',
  },
}
```

Use `style: 'badge'` for the filled pill treatment with background and border, or `style: 'inline'` for a lighter text-first treatment that keeps icon and text on one line without the pill chrome.

`iconPosition` can be `'left'`, `'right'`, or `'only'`.

Additional badge behavior implemented in core:

- `colorMap` and `defaultColor` can be static values or functions of the current entity.
- Badge color objects can use `background`, `text`, `color`, `textColor`, `icon`, `iconColor`, and `iconPosition`.
- Empty values render as a `-` placeholder badge.

### `currency`

```tsx
{
  key: 'amount', label: 'Salary', renderAs: 'currency',
  renderAsOptions: {
    currency: 'USD',
    locale: 'en-US',
    minimumFractionDigits: 0,
    maximumFractionDigits: 2,
    compact: true,
    compactThreshold: 1000,
    showSymbol: true,
    symbolPosition: 'prefix',
    colorize: true,
    positiveColor: '#198754',
    negativeColor: '#dc3545',
    zeroColor: '#6c757d',
    negativeFormat: 'parentheses',
    decimalAlign: true,
  },
}
```

- `currency` and `locale` drive the formatter.
- `minimumFractionDigits` and `maximumFractionDigits` control decimal precision.
- `compact` switches to compact notation once `compactThreshold` is reached.
- `showSymbol` removes the currency symbol when false.
- `symbolPosition` supports a best-effort prefix or suffix adjustment for common symbols.
- `colorize` can tint positive, negative, and zero values with `positiveColor`, `negativeColor`, and `zeroColor`.
- `negativeFormat: 'parentheses'` wraps negative values in parentheses.
- `decimalAlign` keeps the decimal presentation consistent for table cells.

### `progress`

```tsx
{
  key: 'progress', label: 'Progress', renderAs: 'progress',
  renderAsOptions: {
    color: (entity) => entity.progress >= 80 ? '#198754' : '#ffc107',
    backgroundColor: '#e9ecef',
    height: 10,
    showPercentage: true,
    textColor: '#111827',
  },
}
```

- `color` sets the fill color.
- `backgroundColor` sets the track color.
- `height` controls the bar height.
- `showPercentage` toggles the centered percentage label.
- `textColor` controls the percentage label color.

### `rating`

```tsx
{
  key: 'rating', label: 'Rating', renderAs: 'rating',
  renderAsOptions: {
    max: 5,
    color: '#ffd700',
    emptyColor: '#dee2e6',
    size: 18,
    icon: 'star',
  },
}
```

- `max` sets the number of rating icons.
- `size` controls the icon size.
- `color` sets the filled icon color.
- `emptyColor` sets the unfilled icon color.
- `icon` chooses the registered icon name used for each step.

### `link`

```tsx
{
  key: 'url', label: 'Profile', renderAs: 'link',
  renderAsOptions: {
    text: (entity) => `View ${entity.name}`,
    url:  (entity) => `/users/${entity.id}`,
    newTab: true,
    externalWarning: true,
  },
}
```

- `text` overrides the visible label; when omitted, the field value is used.
- `url` overrides the destination; when omitted, the field value is used.
- `newTab` opens the link in a new tab and adds `noopener noreferrer`.
- `externalWarning` adds an external-link indicator when the URL is detected as external.
- Email-like values without an explicit `url` are converted to `mailto:` links.

### `jsonPreview`

```tsx
{
  key: 'metadata', label: 'Metadata', renderAs: 'jsonPreview',
  renderAsOptions: {
    defaultCollapsed: true,
    maxDepth: 3,
  },
}
```

- String values are parsed as JSON when possible.
- Object and array values render as a collapsible tree.
- `defaultCollapsed` starts nested nodes collapsed, while the root stays expanded.
- `maxDepth` truncates deeper nesting.
- Invalid JSON renders an error state instead of throwing.

### `deltaValue`

```tsx
{
  key: 'delta', label: 'Delta', renderAs: 'deltaValue',
  renderAsOptions: {
    trendIndicator: true,
    fractionDigits: 1,
    suffixPercent: true,
    palette: {
      positive: '#198754',
      negative: '#dc3545',
      neutral: '#6c757d',
    },
  },
}
```

- `suffixPercent` appends a percent sign to the formatted value.
- `trendIndicator` toggles the UP/DOWN/FLAT label prefix.
- `fractionDigits` controls the decimal precision.
- `palette` customizes the positive, negative, and neutral colors.

### `sparkTrend`

```tsx
{
  key: 'trend', label: 'Trend', renderAs: 'sparkTrend',
  renderAsOptions: {
    width: 108,
    height: 34,
    strokeWidth: 1.6,
    positiveColor: '#35d4a1',
    negativeColor: '#e67e7e',
    neutralColor: '#6c757d',
    showDeltaLabel: true,
  },
}
```

- Accepts either a numeric delta or a numeric series array/object with a `series` property.
- A parsed series must contain at least two numeric points; otherwise sparkTrend falls back to a generated synthetic series.
- The fallback series is deterministic per entity and is intended as a visual fallback, not as historical data.
- `width` and `height` set the SVG dimensions.
- `strokeWidth` controls the sparkline stroke.
- `positiveColor`, `negativeColor`, and `neutralColor` choose the accent color by trend direction.
- `showDeltaLabel` toggles the numeric label under the chart.

### `scoreMeter`

```tsx
{
  key: 'healthScore', label: 'Health', renderAs: 'scoreMeter',
  renderAsOptions: {
    density: 'regular',
    showValue: true,
    valueAsPercent: false,
    decimals: 0,
    tagPosition: 'top',
    tagAlign: 'left',
    valueAlign: 'right',
    bands: [
      { max: 35, label: 'Critical', color: '#dc3545', background: 'rgba(220, 53, 69, 0.16)' },
      { max: 65, label: 'Watch',    color: '#fd7e14', background: 'rgba(253, 126, 20, 0.16)' },
      { label: 'Healthy',           color: '#198754', background: 'rgba(25, 135, 84, 0.16)' },
    ],
  },
}
```

- Values are clamped to the 0-100 range.
- `density` switches between regular and compact sizing.
- `showValue` toggles numeric score output.
- `valueAsPercent` controls whether the value includes `%`.
- `decimals` controls numeric precision.
- `tagPosition` supports `top` and `bottom`.
- `tagAlign` and `valueAlign` each support `left` or `right`.
- `bands` maps score ranges to labels, colors, and backgrounds.
- If `bands` is omitted, the default bands are `High` for `85-100`, `Good` for `70-84`, `Medium` for `55-69`, and `Low` below `55`.

Alternative compact confidence-style usage:

```tsx
{
  key: 'confidence', label: 'Confidence', renderAs: 'scoreMeter',
  renderAsOptions: {
    density: 'compact',
    valueAsPercent: true,
    tagPosition: 'bottom',
    tagAlign: 'right',
    valueAlign: 'left',
    bands: [
      { min: 85, label: 'High',   color: '#0d6efd' },
      { min: 70, label: 'Good',   color: '#198754' },
      { min: 55, label: 'Medium', color: '#fd7e14' },
      { label: 'Low',             color: '#dc3545' },
    ],
  },
}
```

### `date` / `time` / `datetime` / `timestamp` / `duration`

For full contract details, option guidance, and a live mixed example, see the [Temporal Fields Guide](/core/guides/temporal-fields).

```tsx
{
  key: 'settlementDate', label: 'Settlement Date', type: 'date', renderAs: 'date',
  renderAsOptions: {
    parseMode: 'iso-date',
    locale: 'en-US',
    timezone: 'local',
    formatPreset: 'medium',
  },
}

{
  key: 'checkpointTime', label: 'Checkpoint Time', type: 'time', renderAs: 'time',
  renderAsOptions: {
    parseMode: 'iso-time',
    locale: 'en-US',
    timezone: 'utc',
    showSeconds: true,
    hour12: false,
  },
}

{
  key: 'postedAt', label: 'Posted At', type: 'datetime', renderAs: 'datetime',
  renderAsOptions: {
    parseMode: 'iso-datetime',
    locale: 'en-US',
    timezone: 'utc',
    formatPreset: 'short',
    relativeTime: true,
    relativeDisplay: 'both',
  },
}

{
  key: 'updatedAtEpoch', label: 'Updated', type: 'timestamp', renderAs: 'timestamp',
  renderAsOptions: {
    parseMode: 'epoch-sec',
    locale: 'en-US',
    timezone: 'utc',
    formatPreset: 'short',
  },
}

{
  key: 'processingLatencySec', label: 'Latency', type: 'duration', renderAs: 'duration',
  renderAsOptions: {
    unit: 'sec',
    format: 'humanized',
    precision: 3,
    humanizedUnits: ['day', 'hour', 'minute', 'second'],
  },
}
```

- Temporal renderers are explicit-contract only.
- `parseMode` values: `epoch-ms`, `epoch-sec`, `iso-date`, `iso-datetime`, `iso-time`, `any`.
- `duration.unit` values: `ms`, `sec`, `min`, `hour`, `day`.
- Temporal option surface includes `locale`, `timezone`, `formatPreset`, `formatOptions`, `relativeTime`, `relativeDisplay`, `showSeconds`, `hour12`, and `fallbackText`.
- Duration-specific options include `format`, `humanizedUnits`, `precision`, `showSign`, and `fallbackText`.
- Humanized duration note:
  - `year` and `month` use fixed approximations (`365` days/year, `30` days/month) for elapsed-time display.
  - For calendar-accurate intervals, precompute at the domain layer and render the resulting value.
- Relative-time display modes:
  - `relativeDisplay: 'relative'` (default) shows only relative text.
  - `relativeDisplay: 'both'` shows absolute timestamp with a smaller relative line below.

## Item Layout

`ItemConfig.layout` controls how fields are arranged within each card or row.

### Auto (default)

```tsx
item: { fields: […], layout: { type: 'auto' } }
```

### Flex

```tsx
item: {
  fields: […],
  layout: {
    type: 'flex',
    flex: { direction: 'row', wrap: false, justify: 'flex-start', align: 'center' },
  },
}
```

### CSS Grid with Template Areas

```tsx
item: {
  fields: [
    { key: 'src',   label: 'Photo', type: 'image', imageOptions: { circular: true, width: 48, height: 48 } },
    { key: 'name',  label: 'Name',  showLabel: false },
    { key: 'email', label: 'Email', type: 'email', showLabel: false },
    { key: 'status', label: 'Status', renderAs: 'badge', showLabel: false },
  ],
  layout: {
    type: 'grid',
    grid: {
      columns: '48px minmax(0,1fr) auto',
      gap: '0.5rem 0.75rem',
      areas: [
        '"src name   name"',
        '"src email  status"',
      ],
    },
  },
}
```

### Sections

```tsx
item: {
  fields: [
    { key: 'name', label: 'Name' }, { key: 'email', label: 'Email', type: 'email' },
    { key: 'role', label: 'Role', renderAs: 'badge' }, { key: 'department', label: 'Department' },
  ],
  layout: {
    type: 'sections',
    sections: [
      { title: 'Contact', fields: ['name', 'email'] },
      { title: 'Role',    fields: ['role', 'department'] },
    ],
  },
}
```
