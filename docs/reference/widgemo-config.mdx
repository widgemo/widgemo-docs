# Widgemo Config

`WidgemoConfig` is the central configuration object. It is zone-oriented and mode-driven.

## Shape

| Field | Type | Default | Description |
|---|---|---|---|
| `zones.header` | `ZoneConfig` | ‚Äî | Optional header zone |
| `zones.content` | `ContentConfig` | ‚Äî | **Required.** Content zone (mode, fields, actions, etc.) |
| `zones.footer` | `ZoneConfig` | ‚Äî | Optional footer zone |
| `collapse.initialState` | `'expanded' \| 'collapsed' \| 'fixed'` | `'expanded'` | Widget collapse behaviour |
| `collapse.button` | `boolean` | `true` | Show/hide the collapse toggle |
| `containerFrame` | `{ shell?, border?, shadow?, borderRadius?, borderColor?, borderWidth?, overflow? }` | ‚Äî | Outer shell frame controls |
| `style` | `CSSProperties` | ‚Äî | Inline styles on the root element |
| `theme` | `string` | ‚Äî | Registered theme name |
| `interactions.onEvent` | `(ctx: InteractionContext) => void` | ‚Äî | Global sink for all actions and gestures |
| `devMode` | `boolean \| DevModeConfig` | `false` | Dev inspector overlay (`allowInProduction` opt-in available) |
| `preRender` | `() => void` | ‚Äî | Config-level callback invoked by `Widgemo` before render |

## Header and Footer Zones

`zones.header` and `zones.footer` each accept a `ZoneConfig` object. Both are optional; the content zone is always required.

<WidgemoShowcase
  description="Header with dynamic title + icon, footer with subtitle ‚Äî a typical starting point."
  data={teamData}
  config={{
    zones: {
      header: {
        title:    (data) => `Team (${data.length} members)`,
        subtitle: (data) => `Active: ${data.filter(d => d.status === 'active').length}`,
        icon: { name: 'users', size: 24, color: '#4f46e5' },
        actions: [{ id: 'export', label: 'Export', icon: 'export', placement: 'pinned' }],
      },
      content: {
        mode: 'table',
        item: {
          fields: [
            { key: 'name',   label: 'Name' },
            { key: 'status', label: 'Status', renderAs: 'badge' },
          ],
          layout: { type: 'auto' },
        },
      },
      footer: {
        subtitle: 'Last synced just now',
      },
    },
  }}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
const config = {
  collapse: { initialState: 'expanded', button: true }, // top-level ‚Äî see ¬ß Collapse below
```

For the full `ZoneConfig` shape, all `ZoneLayout` position options (with interactive configuration examples for every variant), icon, and `themeOverrides`, see [Zone Config](/core/reference/zone-config).

## Content Zone

`zones.content` is the only required zone. It owns the data rendering pipeline ‚Äî mode, field definitions, actions, pagination, search, grouping, sorting, filtering, gestures, loading/error states, and sizing.

```tsx
const config = {
  zones: {
    content: {
      mode: 'table',
      item: {
        fields: [
          { key: 'name',   label: 'Name' },
          { key: 'status', label: 'Status', renderAs: 'badge' },
        ],
        layout: { type: 'auto' },
      },
    },
  },
};
```

For the full `ContentConfig` shape and all options, see [Content Config](/core/reference/content-config).

## Collapse

`config.collapse` controls whether the entire widget can be expanded and collapsed. The toggle button renders in the **header zone** ‚Äî a header zone must be present for it to appear.

        mode: 'table',
        item: {
          fields: [
            { key: 'name', label: 'Name' },
            { key: 'role', label: 'Role' },

      footer: {

      },
    },

```tsx
const config = {
  collapse: { initialState: 'collapsed', button: true },
  zones: {
    header: { title: 'Team Directory', subtitle: 'Click to expand' },
    content: { /* ‚Ä¶ */ },
    footer:  { subtitle: 'Last synced just now' },
  },
};
```

| Option | Type | Default | Effect |
|---|---|---|---|
| `initialState` | `'expanded'` \| `'collapsed'` \| `'fixed'` | `'expanded'` | Starting state. `'fixed'` disables the toggle entirely. |
| `button` | `boolean` | `true` | Show or hide the toggle button in the header zone. |

When collapsed, the content zone and footer zone are hidden. The header zone remains visible.

## Container Frame

`containerFrame` controls the visual shell around the entire widget ‚Äî border, shadow, radius, and the optional outer chrome surface.

```tsx
config={{
  containerFrame: {
    shell: 'none',         // strip outer chrome background
    border: 'on',
    borderRadius: 'square',
    borderColor: '#6366f1',
    borderWidth: '2px',
    shadow: 'none',
    overflow: 'hidden',
  },
  zones: { content: { ‚Ä¶ } },
}}
```

### `ContainerFrame` options

| Option | Type | Default | Effect |
|---|---|---|---|
| `shell` | `'on'` \| `'none'` | `'on'` | Show or hide the outer chrome wrapper. When `'none'`, background is transparent and header/footer default to transparent too. |
| `border` | `'on'` \| `'none'` | `'on'` | Show or hide the outer border. |
| `borderRadius` | `'square'` \| `'rounded'` \| number \| string | theme default | Corner radius. `'square'` forces 0; `'rounded'` uses theme default. |
| `shadow` | `'on'` \| `'none'` | theme default | Box shadow on the outer shell. |
| `borderColor` | `string` | theme default | Explicit border color. |
| `borderWidth` | `string \| number` | theme default | Explicit border width. |
| `overflow` | `CSSProperties['overflow']` | ‚Äî | CSS `overflow` on the shell. |

### Shellless mode (`shell: 'none'`)

When `shell: 'none'`, the outer shell surface is disabled while zone internals continue to render normally:

- Outer background is transparent
- Border and shadow are removed
- Header and footer default to transparent backgrounds
- Outer radius defaults to `0` unless explicitly configured
- Outer overflow defaults to `visible` unless explicitly configured
- `zones.content` spacing remains mode-owned

### Borderless vs shellless ‚Äî when to use which

| | Borderless (`border: 'none', shadow: 'none'`) | Shellless (`shell: 'none'`) |
|---|---|---|
| **Outer card background** | Present (uses theme background) | Transparent |
| **Header/footer background** | Present (theme/card surfaces) | Transparent by default |
| **Border line** | Removed | Removed |
| **Outer radius default** | Theme/card default | `0` unless configured |
| **Visual effect** | Flat card, chrome stripped | Blends into host surface ‚Äî no card boundary |
| **Use when** | Fits inside a panel but keeps its own background | Reads as part of the page, no card boundary |

## theme

`theme` sets a named theme registered via `WidgemoThemeProvider` or `widgemoRegistry.registerWidgemoTheme`. Built-in values are `'light'` (default) and `'dark'`.

```tsx
config={{ theme: 'dark', zones: { ‚Ä¶ } }}
```

Theme tokens control the color palette, spacing, and surface appearance across the entire widget. Per-zone overrides use `zones.header.themeOverrides` / `zones.footer.themeOverrides` in `ZoneConfig`.

For the full token reference, registration API, and `WidgemoThemeProvider`, see [Theme API](/core/reference/theme-api).

## `interactions.onEvent`

`interactions.onEvent` is the global interaction sink for all actions and gestures when no local handler is provided. It receives an `InteractionContext` for every action click, item gesture, and zone-action event across the whole widget.

```tsx
const config = {
  interactions: {
    onEvent: (ctx) => {
      console.log(ctx.kind, ctx.interactionId, ctx.entity);
    },
  },
  zones: { content: { ‚Ä¶ } },
};
```

For the full `InteractionContext` shape and `InteractionKind` values, see [Extension API ‚Äî InteractionContext](/core/reference/extension-api#interactioncontext).

For per-gesture config (`zones.content.gestures`), handler flow, and examples, see [Content Config ‚Äî Gestures](/core/reference/content-config#gestures).

## preRender

`preRender` is a config-level callback invoked synchronously before each render. Use it for lightweight side effects ‚Äî analytics events, telemetry, or pre-flight state checks.

```tsx
const config = {
  preRender: () => {
    analytics.track('widget.render');
  },
  zones: { content: { ‚Ä¶ } },
};
```

`preRender` does not receive arguments. It is separate from registry-based lifecycle hooks (`registerWidgemoHook`). For `postRender`, `onItemClick`, `onModeChange`, `onDragDrop`, and the decision guide on when to use each, see [Extension API ‚Äî Lifecycle Hooks](/core/reference/extension-api#lifecycle-hooks).

## devMode

`devMode: true` or `devMode: { enabled: true }` activates the dev inspector overlay ‚Äî a collapsible panel showing the live serialised config. Useful during development.

For a walkthrough of the Inspector UI, button placement rules, the JSON tree, and the interaction notes panel, see [DevMode](/core/concepts/devmode).

```tsx
// Shorthand
config={{ devMode: true, zones: { content: { ‚Ä¶ } } }}

// Full config
config={{
  devMode: {
    enabled: true,
    allowInProduction: false,    // must opt in to show in production builds
    zone: 'auto',                // 'header' | 'content' | 'footer' | 'floating' | 'auto'
    overlay: {
      showHeader: true,
      showDocsLink: 'https://your-docs.example.com',
      showBranding: true,
      excludeFields: ['zones.content.item.fields'],  // dot-path fields to omit
    },
  },
  zones: { content: { ‚Ä¶ } },
}}
```

| Option | Type | Default | Effect |
|---|---|---|---|
| `enabled` | `boolean` | required | Activates the overlay. |
| `allowInProduction` | `boolean` | `false` | Must be explicitly `true` to render in production builds. |
| `zone` | `'header'` \| `'content'` \| `'footer'` \| `'floating'` \| `'auto'` | `'auto'` | Where to attach the inspector toggle icon. `'auto'` uses the fallback hierarchy: header ‚Üí footer ‚Üí floating. |
| `overlay.showHeader` | `boolean` | `true` | Show the overlay header bar with copy button and docs link. |
| `overlay.showDocsLink` | `string \| false` | unset | URL for the docs link in the overlay header. `false` hides it. |
| `overlay.showBranding` | `boolean` | `true` | Show the Widgemo branding footer in the overlay. |
| `overlay.excludeFields` | `string[]` | unset | Dot-path fields to omit from the JSON display (e.g. `'zones.content.item.fields'`). |
