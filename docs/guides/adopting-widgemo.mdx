# Adopting Widgemo

This guide covers adoption patterns for teams integrating widgemo-core into an existing product. For detailed adoption briefs and agent-ready prompts, see the [adoption docs on GitHub](https://github.com/widgemo/widgemo-core/tree/main/docs/adoption).

## Adoption Phases

### Phase 1: Discovery

Run a read-only discovery pass over the codebase. The goal is to produce an Adoption Brief: a source-of-truth document recording each widget's current patterns before any code is touched.

Key questions:

- Where are data tables and card views currently rendered?
- What are the column and field definitions for each view?
- Which views already have sorting, filtering, or pagination?
- Are custom renderers, badges, or status cells in use?

### Phase 2: Adoption Brief

Create one adoption brief per widget. Include:

```md
### Widget: [name]

**Data source**: [where data comes from]
**Current component**: [current table/list component in use]
**Fields**: [field list with type and renderAs equivalent]
**Actions**: [per-item and zone-level actions]
**Mode**: table | grid | carousel | board
**Sorting**: [default sort field and direction]
**Filtering**: [static or user-controlled filters]
**Migration complexity**: low | medium | high
```

### Phase 3: Refactor

Apply widgemo-core incrementally. Prefer a 1:1 replacement for each widget before expanding.

- Keep the data-fetch layer unchanged and pass data directly to `<Widgemo>`.
- Replace layout components in isolation; do not combine with unrelated UI restructuring.
- Start with `mode: 'table'` and `layout: { type: 'auto' }` before customizing.
- Add `devMode: true` in development to validate config shape and field coverage.
- Remove `devMode` before merging.

## Minimum Viable Config

```tsx
import { Widgemo } from '@widgemo/widgemo-core';
import '@widgemo/widgemo-core/dist/widgemo-theme.css';

<Widgemo
	data={rows}
	config={{
		zones: {
			header: { title: 'Widget Title' },
			content: {
				mode: 'table',
				item: {
					fields: [
						{ key: 'id', label: 'ID' },
						{ key: 'name', label: 'Name' },
					],
					layout: { type: 'auto' },
				},
			},
		},
	}}
/>
```

## Agent-Ready Prompts

The `docs/adoption/` folder in widgemo-core includes copy-paste prompts:

| File | Purpose |
| --- | --- |
| `AGENT_PROMPTS_ADOPTION.md` | Discovery, refactor, and validation prompts |
| `ADOPTION_DISCOVERY.md` | Read-only discovery instructions |
| `ADOPTION_REFACTOR.md` | Refactor implementation rules |
| `ADOPTION_BRIEF_TEMPLATE.md` | Source-of-truth brief template |
