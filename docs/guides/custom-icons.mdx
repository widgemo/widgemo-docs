# Custom Icons

Use this page to extend or override icons in the Widgemo icon registry.

If you need the built-in icon catalog, use [Built-in Icons](/core/components/icons/built-in-icons).

## Register a Custom Icon

```tsx
import { widgemoRegistry } from '@widgemo/widgemo-core';
import { CalendarIcon } from '@heroicons/react/24/outline';

widgemoRegistry.registerWidgemoIcon({
	name: 'calendar',
	component: ({ size = 16, className, color = 'currentColor' }) => (
		<CalendarIcon
			width={size}
			height={size}
			className={className}
			style={{ color }}
		/>
	),
});
```

The icon component receives `{ size?, className?, color? }`. Returning `null` falls back to the built-in SVG.

## Built-in Icon Catalog

Use [Built-in Icons](/core/components/icons/built-in-icons) for the current built-in icon names and categories.

## Override a Built-in Icon

Register under the same name to replace it globally:

```tsx
widgemoRegistry.registerWidgemoIcon({
	name: 'edit',
	component: ({ size = 16, color = 'currentColor' }) => (
		<MyCustomEditIcon width={size} style={{ color }} />
	),
});
```

## Collision Behavior

Collision semantics are defined in [Extension API: Registration Collisions](/core/reference/extension-api#registration-collisions).

Use namespaced names (for example, `acme.edit`) to reduce accidental collisions.

See [Extension API: Registration Collisions](/core/reference/extension-api#registration-collisions) for exact behavior.

## Using Icons in Config

Anywhere `icon` accepts a string, provide the registered name:

```tsx
actions: [
	{ id: 'schedule', label: 'Schedule', icon: 'calendar', placement: 'pinned' },
]
```

```tsx
zones: {
	header: {
		icon: { name: 'calendar', size: 24, color: '#4f46e5' },
	},
}
```
