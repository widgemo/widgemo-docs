# DevMode

## What DevMode Is

DevMode is a development-time inspection feature in `widgemo-core`.

When enabled, Widgemo exposes the **Widgemo Inspector** so you can inspect active config and interaction wiring while building your app.

DevMode is active in development by default (`NODE_ENV === 'development'`).

For controlled environments (like docs/sandbox hosts), you can opt in outside development with:

```tsx
const config = {
  devMode: {
    enabled: true,
    allowInProduction: true,
  },
  zones: {
    content: {
      mode: 'table',
      item: { fields: [{ key: 'name' }], layout: { type: 'auto' } },
    },
  },
};
```

## Enabling DevMode

```tsx
const config = {
  devMode: true,
  zones: {
    content: {
      mode: 'table',
      item: { fields: [{ key: 'name' }], layout: { type: 'auto' } },
    },
  },
};

// Disable DevMode
const config = {
  devMode: false,
  zones: {
    content: {
      mode: 'table',
      item: { fields: [{ key: 'name' }], layout: { type: 'auto' } },
    },
  },
};
```

## What Happens When Enabled

- A **Widgemo Inspector** button appears on the widget.
- Selecting it opens the Inspector modal.
- The modal shows your active config in an expandable JSON tree.
- DevMode behavior is development-only by default; use `allowInProduction: true` only in trusted environments.

### Inspector Button Placement

By default, the **Widgemo Inspector** button placement follows this order:

1. If a header zone is present, the Inspector button appears as the **rightmost button in the header zone**.
2. If no header zone is present, but a footer zone is present, it appears as the **rightmost button in the footer zone**.
3. If neither header nor footer is present, the Inspector button **floats at the top-right above the content zone**.

## Inspector Modal Walkthrough

1. Enable `devMode: true` in your config.
2. Render your widget in development mode.
3. Click the **Widgemo Inspector** button.
4. Review the JSON tree and interaction notes.
5. Copy the current inspected config when needed.

## Inspector Components

### Action buttons and what each does

- **Defaults: On/Off**
  - Toggles materialized defaults in the JSON view.
  - When defaults are shown, keys that were added by defaults are emphasized.
- **Expand all / Collapse all**
  - Expands or collapses the full JSON tree.
- **Copy**
  - Copies the currently displayed Inspector JSON.
- **Close (×)**
  - Closes the Inspector modal.

### Interaction notes and what they communicate

- The bottom panel is labeled **Interaction Notes**.
- It summarizes detected interaction wiring (for example global sink and configured handlers).
- Use **Show/Hide** to expand or collapse the details panel.

### JSON view and color coding meaning

The JSON viewer uses token colors to make values easier to scan:

- **Keys**: purple by default.
- **Keys added by defaults toggle**: emphasized blue highlight.
- **Strings**: green and quoted.
- **Numbers**: blue.
- **Booleans**: red.
- **Null**: purple.
- **Object/array brackets and structure markers**: neutral gray.

Keys with metadata also show a dotted underline and tooltip details (type, defaults, and allowed values where available).

### How to use it during development

- Validate that the config currently rendered in your app matches your expectations.
- Expand only the sections you are debugging (zones, interactions, mode config).
- Toggle defaults to distinguish explicit config from implied defaults.
- Use **Copy** to share or diff the inspected config during debugging.
