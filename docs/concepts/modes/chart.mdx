# Chart Mode

`modeConfig.chart` owns the full content area as a chart. Use it when the visualization itself is the primary surface, not a small field renderer. For inline composition bars, see [compositionBar vs Chart Mode](/core/components/renderers/composition-bar-guide#when-to-use-compositionbar-vs-chart-mode).

Five series types are supported: `bar`, `line`, `area`, `pie`, and `donut`. Several options (`color`, `lineThickness`, `showDots`, `areaGradient`, `donutInnerRadiusRatio`) can be set at the top-level chart config as a default, or overridden per series.

## Chart Options Reference

| Option | Values | Default | Effect |
|---|---|---|---|
| `xAxis` | string | `'name'` | Selects the label field used for chart categories. |
| `height` | number | `300` | Sets the chart height in pixels. |
| `showLegend` | `true`, `false` | `true` | Shows or hides the legend. |
| `legendAlign` | `left`, `center`, `right` | `center` | Aligns the legend left, center, or right. |
| `showGrid` | `true`, `false` | `true` | Shows or hides the cartesian grid. |
| `showLabels` | `true`, `false` | `false` | Shows or hides data labels on bars, lines, and polar segments. |
| `colors` | string array | theme tokens | Overrides the default series color palette. |
| `orientation` | `vertical`, `horizontal` | `vertical` | Flips bar charts to horizontal. Only applies when all series are `bar`. |
| `lineThickness` | number (`0.5`â€“`20`) | `3.5` | Sets the default stroke width for line and area series. Can be overridden per series. |
| `showDots` | `true`, `false` | `false` | Shows or hides line/area data points. Can be overridden per series. |
| `areaGradient` | `true`, `false` | `true` | Enables or disables gradient fills for area series. Can be overridden per series. |
| `showTooltip` | `true`, `false` | `true` | Enables or disables hover tooltips. |
| `donutInnerRadiusRatio` | number (`0.2`â€“0.9`) | `0.58` | Controls the donut hole size. Can be overridden per series. |
| `tooltip.position` | `top`, `bottom`, `left`, `right`, `top-right`, `top-left`, `bottom-right`, `bottom-left` | `top-right` | Sets tooltip placement. |
| `tooltip.render` | `(context) => ReactNode` | unset | Replaces the built-in tooltip content. |

### Per-Series Overrides

`color`, `lineThickness`, `showDots`, `areaGradient`, and `donutInnerRadiusRatio` can be set on each individual series item to override the chart-level default for that series only. Here the Revenue series overrides color, thickness, and dots while Visits inherits the chart defaults.

<WidgemoShowcase
  description="Revenue series overrides color, lineThickness, and showDots. Visits inherits the chart-level defaults."
  data={lineChartData}
  config={perSeriesChartConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
chart: {
  showDots: false,           // default for all series
  lineThickness: 1.5,        // default for all series
  series: [
    { key: 'revenue', type: 'line', showDots: true, color: '#6366f1', lineThickness: 3 },  // overrides
    { key: 'visits', type: 'line' },  // inherits chart defaults
  ],
}
```

## Mode-Specific Caveats

- `orientation` applies only when all series are `bar`.
- `donutInnerRadiusRatio` applies only to donut series.
- Only one `pie` or `donut` series is supported per chart.
- Mixing polar (`pie`, `donut`) and cartesian (`bar`, `line`, `area`) series in the same chart is not supported.
- Sorting (`config.sorting`) is not supported in chart mode. Pre-sort your data before passing it in if a specific order is needed.
