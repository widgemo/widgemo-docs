# Carousel Mode

`modeConfig.carousel` controls horizontal browsing behavior for featured content. Use carousel when items are best consumed as a swipeable sequence instead of a dense grid or table.

## Shared Behavior

Pagination, search, static filtering, actions, groupings, and gestures work the same as other modes — see the reference pages:

- [Content Config: Pagination and Search](/core/reference/content-config#pagination-and-search)
- [Content Config: Static Filtering](/core/reference/content-config#static-filtering-and-sorting)
- [Content Config: Actions and Interactions](/core/reference/content-config#actions)
- [Content Config: Grouping](/core/reference/content-config#grouping)
- [Content Config: Shape](/core/reference/content-config#shape) for responsive, loading, and error fields
- [Action Config](/core/reference/action-config) for action placement and overflow behavior

## Grouping

`zones.content.groupings` works with carousel mode. Each group renders as a separate labeled carousel with an optional collapse control.

<WidgemoShowcase
  description="Items grouped by team, each group rendering as its own collapsible carousel."
  data={carouselGroupingData}
  config={carouselGroupingConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
zones: {
  content: {
    groupings: [
      {
        fieldKey: 'team',
        collapsible: true,
        renderer: (groupValue, count) => `${groupValue} · ${count} items`,
      },
    ],
  },
}
```
