# Grid Mode

`modeConfig.grid` tiles items into a responsive card grid. Use grid when card-based browsing matters more than dense row scanning.

## Shared Behavior

Pagination, search, filtering, sorting, actions, and grouping are shared across modes — see the reference pages:

- [Content Config: Pagination and Search](/core/reference/content-config#pagination-and-search)
- [Content Config: Static Filtering and Sorting](/core/reference/content-config#static-filtering-and-sorting)
- [Content Config: Actions and Interactions](/core/reference/content-config#actions-and-interactions)
- [Content Config: Grouping](/core/reference/content-config#grouping)
- [Content Config: Shape](/core/reference/content-config#shape) for responsive, loading, and error fields
- [Field Config](/core/reference/field-config) for item-level field rendering and layout rules

### Masonry Layout

Masonry packing is most visible when card heights vary naturally. Here each card contains a single image at its natural aspect ratio.

<WidgemoShowcase
  description="Uses layoutMode='masonry' with cards of varying heights to produce a waterfall layout."
  data={gridMasonryData}
  config={gridDensePackingConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
modeConfig: {
  grid: {
    layoutMode: 'masonry',
    minItemWidth: '180px',
    maxColumns: 4,
    gap: '0.75rem',
  },
}
// Image field — natural aspect ratios drive card heights:
item: {
  fields: [
    {
      key: 'imageUrl',
      label: '',
      type: 'image',
      imageOptions: { width: '100%', height: 'auto', objectFit: 'cover', borderRadius: 6 },
    },
  ],
}
```

### Grouping in Grid

`zones.content.groupings` also applies to grid mode. In grid, grouping controls are rendered as grouped sections (not table header controls).

## Grid Options Reference

| Option | Values | Default | Effect |
|---|---|---|---|
| `layoutMode` | `standard`, `masonry` | `standard` | Selects regular CSS grid or masonry layout strategy. |
| `gap` | any CSS length | `1rem` | Sets spacing between cards. |
| `minItemWidth` | any CSS length | `220px` | Sets minimum card width before wrapping. |
| `maxColumns` | any number >= 1 | unset | Caps the maximum number of columns. |
| `autoFlow` | `row`, `column`, `dense` | `row` | Controls packing direction and density behavior. |
| `justifyItems` | `start`, `end`, `center`, `stretch` | browser default | Controls horizontal card alignment inside each grid cell. |
| `alignItems` | `start`, `end`, `center`, `stretch` | browser default | Controls vertical card alignment inside each grid cell. |

For grouping contract details and mode applicability, see [Content Config: Grouping](/core/reference/content-config#grouping).

For item-level card layout/field rendering details, see [Field Config](/core/reference/field-config).

## Mode-Specific Caveats

- `autoFlow: dense` can visually reorder packing to fill gaps; use with care if strict visual sequence matters.
- Grouping in grid renders grouped sections, not table-style header controls.

## Coverage Summary

Every grid setting is represented by at least one live example on this page:

- `minItemWidth`, `maxColumns`, `gap`: Responsive Grid (Baseline)
- `layoutMode`: Masonry Layout
- `autoFlow`: documented in Grid Option Reference (best visualized when card footprints vary)
- `justifyItems`, `alignItems`: Alignment Controls
- `groupings` in grid mode: Grouped Grid
- shared content settings with grid (`search`, `pagination`, `filtering`, `sorting`, `actions`, `gestures`): Shared-Feature Boundary links above
