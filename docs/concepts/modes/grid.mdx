# Grid Mode

`modeConfig.grid` tiles items into a responsive card grid. Use grid when card-based browsing matters more than dense row scanning.

## Shared Behavior

Pagination, search, static filtering, actions, gestures, and groupings work the same as other modes — see the reference pages:

- [Content Config: Pagination and Search](/core/reference/content-config#pagination-and-search)
- [Content Config: Static Filtering](/core/reference/content-config#static-filtering-and-sorting)
- [Content Config: Actions and Interactions](/core/reference/content-config#actions)
- [Content Config: Grouping](/core/reference/content-config#grouping)
- [Content Config: Shape](/core/reference/content-config#shape) for responsive, loading, and error fields
- [Item Config](/core/reference/item-config) for item-level shape and layout rules
- [Field Config](/core/reference/field-config) for field-level rendering schema and renderer options

### Masonry Layout

Masonry packing is most visible when card heights vary naturally. Here each card contains a single image at its natural aspect ratio.

<WidgemoShowcase
  description="Uses layoutMode='masonry' with cards of varying heights to produce a waterfall layout."
  data={gridMasonryData}
  config={gridDensePackingConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
modeConfig: {
  grid: {
    layoutMode: 'masonry',
    minItemWidth: '180px',
    maxColumns: 4,
    gap: '0.75rem',
  },
}
// Image field — natural aspect ratios drive card heights:
item: {
  fields: [
    {
      key: 'imageUrl',
      label: '',
      type: 'image',
      imageOptions: { width: '100%', height: 'auto', objectFit: 'cover', borderRadius: 6 },
    },
  ],
}
```

<div style={{ marginBottom: '1rem' }} />

### Grouping in Grid

`zones.content.groupings` also applies to grid mode. In grid, grouping controls are rendered as grouped sections (not table header controls).

<WidgemoShowcase
  description="Groups cards by team with collapsible group sections and a custom group label renderer."
  data={gridModeData}
  config={gridGroupingConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
zones: {
  content: {
    groupings: [
      {
        fieldKey: 'team',
        collapsible: true,
        renderer: (groupValue, count, isCollapsed) =>
          `${groupValue} team · ${count} members ${isCollapsed ? '▶' : '▼'}`,
      },
    ],
  },
}
```

<div style={{ marginBottom: '1rem' }} />

### Search and Filtering

A search bar renders above the grid when `config.search` is enabled. Static filtering pre-excludes records before render — the grid simply shows fewer cards. This example pre-excludes inactive members and adds a live search over name, role, and team.

<WidgemoShowcase
  description="Search bar above the card grid with a static filter pre-applied to exclude inactive members."
  data={gridModeData}
  config={gridSearchConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
zones: {
  content: {
    search: {
      enabled: true,
      fields: ['name', 'role', 'team'],
      placeholder: 'Search members...',
    },
    filtering: [
      { fieldKey: 'status', operator: 'ne', value: 'inactive' },
    ],
  },
}
```

<div style={{ marginBottom: '1rem' }} />
