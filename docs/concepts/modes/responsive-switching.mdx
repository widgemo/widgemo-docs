# Responsive Switching

Responsive switching uses a base `content.mode` and then swaps in breakpoint-specific mode and `modeConfig` overrides from `content.responsive.breakpoints`.

Shared options such as search, filtering, pagination, actions, and gestures are covered in [Content Config](/core/reference/content-config).

## What It Changes

Use responsive switching when the same dataset should present differently across widths.

- `content.mode` defines the default desktop mode.
- `content.responsive.breakpoints` adds width-specific overrides.
- Each breakpoint can swap the mode and tune that mode's config.

```tsx
const config = {
  zones: {
    content: {
      mode: 'table',
      item: {
        fields: [
          { key: 'name', label: 'Name' },
          { key: 'department', label: 'Department' },
          { key: 'status', label: 'Status', renderAs: 'badge' },
        ],
        layout: { type: 'auto' },
      },
      modeConfig: {
        table: {
          showHeader: true,
          hover: true,
        },
      },
      responsive: {
        breakpoints: {
          tablet: {
            mode: 'grid',
            modeConfig: {
              grid: {
                minItemWidth: '260px',
                gap: '0.75rem',
              },
            },
          },
          mobile: {
            mode: 'carousel',
            modeConfig: {
              carousel: {
                itemWidth: 260,
                showIndicators: true,
              },
            },
          },
        },
      },
    },
  },
};
```

## Responsive Options

| Option | What it changes |
|---|---|
| `content.mode` | Sets the default mode before breakpoints are applied. |
| `content.responsive.breakpoints` | Declares width-specific overrides. |
| `breakpoints.<name>.mode` | Switches the mode at that breakpoint. |
| `breakpoints.<name>.modeConfig` | Overrides the chosen mode's config at that breakpoint. |
