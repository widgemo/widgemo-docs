# Responsive Switching

## Mode Purpose and When to Use

Responsive switching uses a base `content.mode` and then swaps in breakpoint-specific mode and `modeConfig` overrides from `content.responsive.breakpoints`.

Use responsive switching when the same dataset should present differently at desktop, tablet, and mobile widths without duplicating content contracts.

## Core Examples

### What It Changes

Use responsive switching when the same dataset should present differently across widths.

- `content.mode` defines the default desktop mode.
- `content.responsive.breakpoints` adds width-specific overrides.
- Each breakpoint can swap the mode and tune that mode's config.

```tsx
const config = {
  zones: {
    content: {
      mode: 'table',
      item: {
        fields: [
          { key: 'name', label: 'Name' },
          { key: 'department', label: 'Department' },
          { key: 'status', label: 'Status', renderAs: 'badge' },
        ],
        layout: { type: 'auto' },
      },
      modeConfig: {
        table: {
          showHeader: true,
          hover: true,
        },
      },
      responsive: {
        breakpoints: {
          tablet: {
            mode: 'grid',
            modeConfig: {
              grid: {
                minItemWidth: '260px',
                gap: '0.75rem',
              },
            },
          },
          mobile: {
            mode: 'carousel',
            modeConfig: {
              carousel: {
                itemWidth: 260,
                showIndicators: true,
              },
            },
          },
        },
      },
    },
  },
};
```

## Mode Option Reference

| Option | Values | Default | Effect |
|---|---|---|---|
| `content.mode` | `table`, `grid`, `board`, `carousel`, `chart`, string | required | Sets the default mode before breakpoints are applied. |
| `content.responsive.breakpoints` | record of breakpoint objects | unset | Declares width-specific overrides. |
| `breakpoints.<name>.mode` | `table`, `grid`, `board`, `carousel`, `chart`, string | unset | Switches the mode at that breakpoint. |
| `breakpoints.<name>.modeConfig` | partial `ModeConfig` | unset | Overrides the chosen mode's config at that breakpoint. |

## Mode-Specific Caveats

- Each breakpoint `modeConfig` should match the mode selected at that breakpoint.
- Shared contracts such as actions, search, and filtering still come from `zones.content` and are not redefined per mode page.
