# Theming

widgemo-core supports multiple theming approaches so you can match your product vision or fit seamlessly into an existing app style.

widgemo-core includes a built-in default theme out of the box, and then supports deeper theming when you need more control.

## Built-in Default Theme

If you do not provide a custom theme object, Widgemo uses built-in theme tokens and styles.

- With `WidgemoThemeProvider`, the default is `theme="light"`.
- With `theme="dark"`, the built-in dark preset is applied.

### Example Widget (Default Light)

<WidgemoThemeProvider theme="light">
  <div className="widgemo-showcase__widget">
    <Widgemo data={themingExampleData} config={baseThemeConfig} />
  </div>
</WidgemoThemeProvider>

  </div>
</WidgemoThemeProvider>

### Example Code

```tsx
<WidgemoThemeProvider theme="light">
  <Widgemo data={data} config={config} />
</WidgemoThemeProvider>

<WidgemoThemeProvider theme="dark">
  <Widgemo data={data} config={config} />
</WidgemoThemeProvider>
```

After the built-in defaults, you can customize theming in three ways:

1. `WidgemoThemeProvider` preset (`light`, `dark`, `auto`)
2. `WidgemoThemeProvider` custom theme object (`colors`, `spacing`, `zone`)
3. Per-zone `themeOverrides` in `config`

For full token lists, see [Theme API](/core/reference/theme-api) and [CSS Variables](/core/reference/css-variables).

## 1. Theme Provider Presets

Use preset themes when you want fast parity with your app mode (`light`, `dark`, `auto`) without defining custom tokens.

This option keeps setup lightweight and works well when your app already supports light and dark modes.

### Example Widget

<WidgemoThemeProvider theme="dark">
  <div className="widgemo-showcase__widget">
    <Widgemo data={themingExampleData} config={baseThemeConfig} />
  </div>
</WidgemoThemeProvider>

<div style={{ marginBottom: '1rem' }} />

### Example Code

```tsx
import { WidgemoThemeProvider } from '@widgemo/widgemo-core';

<WidgemoThemeProvider theme="dark">
  <Widgemo data={data} config={config} />
</WidgemoThemeProvider>

<WidgemoThemeProvider theme="auto">
  <Widgemo data={data} config={config} />
</WidgemoThemeProvider>
```

`theme="auto"` is evaluated at render time from `window.matchMedia('(prefers-color-scheme: dark)')` and does not subscribe to live OS preference changes.

## 2. Custom Theme Object

Use a custom theme object when you want consistent branded tokens across all widgets.

This gives you one central place to apply brand colors, spacing, and shared visual rules across the entire widget surface.

### Example Widget

<WidgemoThemeProvider theme={customTheme}>
  <div className="widgemo-showcase__widget">
    <Widgemo data={themingExampleData} config={baseThemeConfig} />
  </div>
</WidgemoThemeProvider>

<div style={{ marginBottom: '1rem' }} />

### Example Code

```tsx
<WidgemoThemeProvider theme={{
  colors: {
    actionButtonBg: '#0f172a',
    actionButtonColor: '#f8fafc',
    actionButtonHoverBg: '#1e293b',
  },
  spacing: {
    'border-radius': '10px',
    'card-padding': '1.25rem',
  },
  zone: {
    backgroundColor: '#1e293b',
    titleColor: '#f8fafc',
    subtitleColor: '#94a3b8',
    padding: '1rem 1.25rem',
  },
}}>
  <Widgemo data={data} config={config} />
</WidgemoThemeProvider>
```

## 3. Per-Zone Theme Overrides

Use `themeOverrides` when you want targeted visual changes for a specific header/content/footer zone without introducing a full global theme object.

Use this when only part of a widget needs special styling and the rest should continue using the current theme.

<WidgemoShowcase title="Example Widget" data={themingExampleData} config={zoneOverrideConfig} />

<div style={{ marginBottom: '1rem' }} />

### Example Code

```tsx
const config = {
  zones: {
    header: {
      title: 'Styled Header',
      themeOverrides: {
        backgroundColor: '#111827',
        titleColor: '#f9fafb',
        subtitleColor: '#9ca3af',
      },
    },
  },
};
```

## Runtime Theme Precedence

Theme values apply from highest to lowest priority:

1. Inline zone/content overrides (`config.style`, `zones.*.themeOverrides`, `zones.*.style`)
2. Runtime zone theme merge (`registry theme` + provider `theme.zone`)
3. Provider variables (`theme.colors`, `theme.spacing`)
4. Root fallback CSS variables from library stylesheet

Use `WidgemoThemeProvider` for global consistency and `themeOverrides` for local exceptions.

## Accessing Theme in Custom Components

```tsx
function MyModeHeader() {
  const theme = useWidgemoTheme();
  return <header style={{ background: theme.zone?.backgroundColor }} />;
}
```

Without a provider, `useWidgemoTheme()` returns `{}`.
