## Shared vs Mode-Specific

Mode guides are intentionally mode-specific: they cover settings under `modeConfig.<mode>` and mode-only caveats.

Shared behavior is canonical in Reference and should be treated as the source of truth:

- [Content Config](/core/reference/content-config): search, pagination, filtering, sorting, grouping, gestures, responsive breakpoints, loading, and error handling
- [Action Config](/core/reference/action-config): action shape, placement, and overflow behavior
- [Field Config](/core/reference/field-config): item field rendering and layout contract
- [Widgemo Config](/core/reference/widgemo-config): top-level widget configuration surface
- [Content Config: Shape](/core/reference/content-config#shape): canonical location for shared `zones.content` contract fields
