# Configuration Model

Widgemo configuration is easiest to understand as four nested layers:

1. `Widgemo` props: runtime inputs such as `data`, `config`, `loading`, and `error`
2. `zones`: page regions such as `header`, `content`, and `footer`
3. `content`: how data is rendered (`mode`, filtering, sorting, gestures, pagination)
4. `item` and `fields`: how one entity is laid out and how each value is displayed

This is the core mental model for the library.

- `zones` answer: where does this UI live?
- `content` answers: how is the dataset rendered?
- `item` answers: how is one record arranged?
- `fields` answer: how does each value appear?

## The Four Layers

## 1. Widgemo Props

At the outermost level, you pass `data` and `config` into `<Widgemo>`.

```tsx
<Widgemo data={rows} config={config} />
```

This is the runtime shell. Everything else lives inside `config`.

## 2. Zones

Zones organize the widget surface into named regions.

```tsx
config = {
  zones: {
    header: { title: 'Accounts' },
    content: { ... },
    footer: { subtitle: 'Last synced 2m ago' },
  },
};
```

Use zones when you want structure such as titles, actions, summaries, and supporting metadata around the main content.

## 3. Content

`zones.content` defines how the dataset behaves and renders.

```tsx
content: {
  mode: 'table',
  filtering: [...],
  sorting: [...],
  pagination: { pageSize: 20 },
  item: { ... },
}
```

This is where you choose the rendering mode and content-level behavior.

## 4. Item and Fields

`item` defines how a single entity is rendered. `fields` define each displayed value.

```tsx
item: {
  fields: [
    { key: 'name', label: 'Name' },
    { key: 'status', label: 'Status', renderAs: 'badge' },
  ],
  layout: { type: 'auto' },
}
```

This is where you decide field order, renderers, labels, widths, and item layout.

## Practical Reading Order

When reading a Widgemo config, use this order:

1. Start with `zones` to understand the high-level shell
2. Read `zones.content.mode` to understand the main rendering strategy
3. Read `zones.content.item.layout` to understand per-record layout
4. Read `zones.content.item.fields` to understand the actual values shown
