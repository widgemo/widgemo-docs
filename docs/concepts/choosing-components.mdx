# Choosing Components

Use this page to decide whether your use case should start with a mode, a field type, or a renderer.

## Decision Flow

1. Need a whole-surface layout change for a dataset?
- Use a mode under [Components Overview](/core/components/overview).

2. Need a domain-level data behavior contract reused across many configs?
- Prefer field type semantics in [Field Components Overview](/core/components/fields/overview).

3. Need a display-only visual treatment for an existing field?
- Use a renderer from [Renderer Components Overview](/core/components/renderers/overview).

## Modes vs Renderers

- Modes control the full content surface (`table`, `grid`, `carousel`, `board`, `chart`, responsive switching).
- Renderers control how one field value is displayed (`badge`, `currency`, `progress`, and more).

## type vs renderAs

- Choose `type` first to express domain semantics.
- Add `renderAs` when you need a specific visual treatment.
- When both are present, `renderAs` determines display output.
