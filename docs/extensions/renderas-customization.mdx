# Custom renderAs

A custom `renderAs` renderer is the lightest extension point in Widgemo.

Use it when you want to change how a single value is displayed inside an otherwise normal field.

## 1. Create the renderer

```tsx
// src/renderers/JsonRenderer.tsx
import React from 'react';
import type { RenderAsRegistryEntry } from '@widgemo/widgemo-core';

export const jsonRenderer: RenderAsRegistryEntry = {
  name: 'json',
  render: (value, _options, _entity) => (
    <pre style={{ fontSize: '0.75rem', margin: 0, overflowX: 'auto' }}>
      {JSON.stringify(value, null, 2)}
    </pre>
  ),
  defaultOptions: {},
};
```

## 2. Register at app boot

```tsx
import { widgemoRegistry } from '@widgemo/widgemo-core';
import { jsonRenderer } from './renderers/JsonRenderer';

widgemoRegistry.registerWidgemoRenderAs(jsonRenderer);
```

## 3. Use in any field

```tsx
const fields = [
  { key: 'id', label: 'ID' },
  { key: 'name', label: 'Name' },
  { key: 'metadata', label: 'Metadata', renderAs: 'json' },
];
```

## When to Choose Custom renderAs

Use custom `renderAs` when:

- the field is still conceptually a normal field
- you only need a different visual treatment
- you do not need a brand-new content mode
- you do not need a new first-class field type contract
