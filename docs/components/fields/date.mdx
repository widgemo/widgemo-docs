# date Field Type

`type: 'date'` marks a field as a calendar date value. Pair it with `renderAs: 'date'` and a `parseMode` to render it in a locale-aware, human-readable format. Without `renderAs`, the raw value is displayed as plain text.

## Core Examples

### Basic Date Rendering

<WidgemoShowcase
  description="ISO date string rendered with parseMode: 'iso-date' and formatPreset: 'medium'."
  data={dateData}
  config={dateBasicConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'dueDate',
  label: 'Due Date',
  type: 'date',
  renderAs: 'date',
  renderAsOptions: { parseMode: 'iso-date', locale: 'en-US', formatPreset: 'medium' },
}
```

<div style={{ marginBottom: '1rem' }} />

### Format Presets

`formatPreset` controls how much detail is shown. All presets use `Intl.DateTimeFormat` internally — output adapts to `locale`.

<WidgemoShowcase
  description="The same date rendered with short, medium, long, and full presets."
  data={dateData}
  config={datePresetsConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{ renderAsOptions: { parseMode: 'iso-date', locale: 'en-US', formatPreset: 'short' } }
// → 5/29/26

{ renderAsOptions: { parseMode: 'iso-date', locale: 'en-US', formatPreset: 'medium' } }
// → May 29, 2026  (default when formatPreset is omitted)

{ renderAsOptions: { parseMode: 'iso-date', locale: 'en-US', formatPreset: 'long' } }
// → May 29, 2026

{ renderAsOptions: { parseMode: 'iso-date', locale: 'en-US', formatPreset: 'full' } }
// → Friday, May 29, 2026
```

<div style={{ marginBottom: '1rem' }} />

### Relative Time

`relativeTime: true` shows human-friendly relative output ("2 weeks ago"). `relativeDisplay: 'both'` stacks the absolute date above the relative label.

## Field Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `key` | string | required | Entity property to read the date value from. |
| `label` | string | unset | Column header or card label. |
| `type` | `'date'` | — | Marks the field as a calendar date. Pair with `renderAs: 'date'` for formatted output. |
| `renderAs` | `'date'` | unset | Required for locale-aware date rendering. Without it, raw value renders as plain text. |
| `condition` | `(entity) => boolean` | unset | Hides the field for rows where the function returns `false`. |
| `sortable` | `boolean` | `false` | Enables column sorting in table mode. Sorts chronologically when `parseMode` is set. |
| `width` | CSS length or number | unset | Sets column width in table mode. |
| `align` | `'left'`, `'center'`, `'right'` | `'left'` | Text alignment in table mode. |

### `renderAsOptions` for `renderAs: 'date'`

| Option | Type | Default | Effect |
|---|---|---|---|
| `parseMode` | `'iso-date'` \| `'epoch-ms'` \| `'epoch-sec'` \| `'iso-datetime'` \| `'any'` | **required** | Declares the source format. Without this, renders `'Invalid'`. |
| `locale` | string | `'en-US'` | BCP 47 locale tag passed to `Intl.DateTimeFormat`. |
| `timezone` | `'local'` \| `'utc'` \| IANA string | `'local'` | Timezone for display. Does not affect the stored value. |
| `formatPreset` | `'short'` \| `'medium'` \| `'long'` \| `'full'` | `'medium'` | Controls date detail level. |
| `formatOptions` | `Intl.DateTimeFormatOptions` | unset | Direct override — merged with preset, takes precedence. |
| `relativeTime` | `boolean` | `false` | Show relative time ("2 weeks ago") instead of absolute. |
| `relativeDisplay` | `'relative'` \| `'both'` | `'relative'` | When `relativeTime: true`: show only relative, or absolute+relative stacked. |
| `fallbackText` | string | `'Invalid'` | Text shown when the value cannot be parsed. |

## Caveats

- `parseMode` is **required**. Without it, the field renders `'Invalid'` regardless of the value.
- `parseMode: 'iso-date'` expects `YYYY-MM-DD`. An ISO datetime string like `2026-05-29T08:00:00Z` will parse correctly with `'iso-datetime'` or `'any'`, not `'iso-date'`.
- `timezone` controls the **display timezone**, not parsing. The underlying `Date` object is always UTC-based.
- Table mode sorts chronologically when `parseMode` is declared and `sortable: true` is set. Without `parseMode`, sort falls back to generic string comparison.
- `relativeTime` uses `Intl.RelativeTimeFormat` — output is locale-sensitive.
