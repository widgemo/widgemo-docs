# duration Field Type

`type: 'duration'` marks a field as an elapsed-time value. Pair it with `renderAs: 'duration'` and a `unit` that matches the source value's magnitude. Three output formats are available: `clock` (`HH:MM:SS`), `humanized` ("2 hours 3 minutes"), and `decimal` (61.25).

## Core Examples

### Clock Format

`format: 'clock'` renders duration as `HH:MM:SS`. Useful for timer-style displays and processing metrics.

<WidgemoShowcase
  description="Processing latency in seconds rendered as HH:MM:SS clock format."
  data={durationData}
  config={durationClockConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'latencySec',
  label: 'Latency',
  type: 'duration',
  renderAs: 'duration',
  renderAsOptions: { unit: 'sec', format: 'clock' },
}
```

<div style={{ marginBottom: '1rem' }} />

### Humanized Format

`format: 'humanized'` renders a readable breakdown ("48 seconds", "2 hours 8 minutes"). `humanizedUnits` controls which time units to include. `precision` limits how many units appear.

<WidgemoShowcase
  description="Left: default humanizedUnits (day/hour/minute/second), precision 2. Right: explicit millisecond-level units, precision 3."
  data={durationData}
  config={durationHumanizedConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
// Default units â€” day, hour, minute, second
{ renderAsOptions: { unit: 'ms', format: 'humanized', precision: 2, showSign: false } }

// Explicit units including milliseconds
{
  renderAsOptions: {
    unit: 'ms',
    format: 'humanized',
    humanizedUnits: ['hour', 'minute', 'second', 'millisecond'],
    precision: 3,
    showSign: false,
  },
}
```

<div style={{ marginBottom: '1rem' }} />

### Decimal Format

`format: 'decimal'` renders the duration as a decimal number in the declared `unit`. Useful for dashboards and finance-style KPIs. `showSign: true` prefixes positive values with `+` and negative with `-`.

## Field Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `key` | string | required | Entity property to read the duration value from. |
| `label` | string | unset | Column header or card label. |
| `type` | `'duration'` | â€” | Marks the field as an elapsed-time value. Pair with `renderAs: 'duration'`. |
| `renderAs` | `'duration'` | unset | Required for formatted output. Without it, raw value renders as plain text. |
| `condition` | `(entity) => boolean` | unset | Hides the field for rows where the function returns `false`. |
| `sortable` | `boolean` | `false` | Enables column sorting. Sorts numerically when `unit` is set. |
| `width` | CSS length or number | unset | Sets column width in table mode. |
| `align` | `'left'`, `'center'`, `'right'` | `'left'` | Text alignment in table mode. |

### `renderAsOptions` for `renderAs: 'duration'`

| Option | Type | Default | Effect |
|---|---|---|---|
| `unit` | `'ms'` \| `'sec'` \| `'min'` \| `'hour'` \| `'day'` | **required** | Declares the source unit of the raw numeric value. |
| `format` | `'clock'` \| `'humanized'` \| `'decimal'` | `'humanized'` | Output format. |
| `humanizedUnits` | Array of `'year'` \| `'month'` \| `'day'` \| `'hour'` \| `'minute'` \| `'second'` \| `'millisecond'` | `['day','hour','minute','second']` | Which units to include in humanized output. Ordered largest-to-smallest. |
| `precision` | `number` | `2` | Max number of units shown (humanized) or decimal places (decimal format). |
| `showSign` | `boolean` | `true` | Prefix positive values with `+` and negative with `-` in decimal/clock formats. |
| `fallbackText` | string | `'Invalid'` | Text shown when the value cannot be parsed. |

## Caveats

- `unit` is **required**. Without it, the field renders `'Invalid'`.
- `format: 'humanized'` with `year` or `month` units uses approximations: `1 year = 365 days`, `1 month = 30 days`. For calendar-accurate intervals, compute the display string in the host app and pass it as `type: 'text'`.
- `showSign` defaults to `true` â€” this prefixes all positive values with `+`. Set `showSign: false` if you don't want sign indicators on positive values.
- `format: 'decimal'` renders the value in the declared `unit` â€” a 3661-second value with `unit: 'sec'` and `precision: 2` renders `3661.00`, not `1.02 hours`. Convert units in the host app or use `humanized` format for multi-unit breakdown.
- Table sorting on duration columns works numerically when `unit` is set and `sortable: true`.
