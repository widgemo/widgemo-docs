# text Field Type

Use `type: 'text'` for general string content without specialized semantic behavior.

## Typical Use Cases

- Names and labels
- Freeform descriptions
- Default fallback for unknown string-like values

## Example

```tsx
{ key: 'name', label: 'Name', type: 'text' }
```
