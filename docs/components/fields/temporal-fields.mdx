# Temporal Fields Guide

Widgemo temporal rendering is explicit-contract by design.

- `date`, `time`, `datetime`, and `timestamp` require `renderAsOptions.parseMode`.
- `duration` requires `renderAsOptions.unit`.
- There is no implicit source-format fallback contract.

## Why This Contract Exists

Different host systems store temporal values differently:

- Unix epoch milliseconds
- Unix epoch seconds
- ISO date strings (`YYYY-MM-DD`)
- ISO datetime strings (`YYYY-MM-DDTHH:mm:ssZ`)
- ISO time strings (`HH:mm:ss`)

Making the source format explicit prevents silent mis-parsing and sorting bugs.

## Complete Option Reference

`date` / `time` / `datetime` / `timestamp` shared options:

- `parseMode` (`epoch-ms` | `epoch-sec` | `iso-date` | `iso-datetime` | `iso-time` | `any`) is required for `date`/`time`/`datetime`/`timestamp`.
- `locale` (`string`)
- `timezone` (`local` | `utc` | IANA timezone string)
- `formatPreset` (`short` | `medium` | `long` | `full`)
- `formatOptions` (`Intl.DateTimeFormatOptions`)
- `relativeTime` (`boolean`)
- `relativeDisplay` (`relative` | `both`) when `relativeTime` is enabled
- `fallbackText` (`string`)

`time` / `datetime` / `timestamp` time-format options:

- `showSeconds` (`boolean`)
- `hour12` (`boolean`)

`duration` options:

- `unit` (`ms` | `sec` | `min` | `hour` | `day`) required
- `format` (`clock` | `humanized` | `decimal`)
- `humanizedUnits` (`year` | `month` | `day` | `hour` | `minute` | `second` | `millisecond`) for `humanized` output
- `precision` (`number`)
- `showSign` (`boolean`)
- `fallbackText` (`string`)

### Duration Best Practices

- For elapsed-runtime UX, `humanizedUnits` is ideal because apps can choose readability level per surface.
- `year` and `month` are approximation units in humanized duration output:
  - `1 year = 365 days`
  - `1 month = 30 days`
- For calendar-accurate business intervals, prefer domain-level calendar calculations before passing values into `renderAs: 'duration'`.
- For predictable UI, explicitly set `humanizedUnits` instead of relying on defaults.

## Valid Contract Values

`parseMode`:

- `epoch-ms`
- `epoch-sec`
- `iso-date`
- `iso-datetime`
- `iso-time`
- `any`

`duration.unit`:

- `ms`
- `sec`
- `min`
- `hour`
- `day`

## Sorting Behavior

Table sorting for temporal fields is contract-aware.

- Temporal columns sort chronologically only when a valid `parseMode` is declared.
- Duration columns sort numerically only when `unit` is declared.
- If no contract is declared, Widgemo falls back to generic comparison semantics.

## Migration Checklist

1. Locate all `type: 'date'` fields and add `renderAs: 'date'` with `parseMode`.
2. For epoch values, switch to `type: 'timestamp'` and set `parseMode` accordingly.
3. Add `unit` for every `type: 'duration'` field.
4. Validate sorting on each temporal column in table mode.
