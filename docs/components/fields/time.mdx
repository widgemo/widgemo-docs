# time Field Type

`type: 'time'` marks a field as a time-of-day value. Pair it with `renderAs: 'time'` and `parseMode: 'iso-time'` to render it in a locale-aware format. Use `showSeconds` and `hour12` to control display granularity.

## Core Examples

### Basic Time Rendering

  description="ISO time string rendered in 24-hour format with seconds."
  data={timeData}

```tsx
{
  key: 'checkpointTime',
  label: 'Checkpoint',
  type: 'time',
  renderAs: 'time',
  renderAsOptions: { parseMode: 'iso-time', locale: 'en-US', showSeconds: true, hour12: false },
}
```

### 12h vs 24h and Seconds

`hour12` toggles 12h/24h format. `showSeconds` adds or removes the seconds component.

<WidgemoShowcase
  description="Same time value rendered in 24h with seconds, 12h without seconds, and a separate scheduled time field."
  data={timeData}
  config={timeFormattingConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
// 24-hour with seconds
{ renderAsOptions: { parseMode: 'iso-time', showSeconds: true, hour12: false } }
// → 08:15:10

// 12-hour without seconds
{ renderAsOptions: { parseMode: 'iso-time', showSeconds: false, hour12: true } }
```


## Field Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `key` | string | required | Entity property to read the time value from. |
| `label` | string | unset | Column header or card label. |
| `type` | `'time'` | — | Marks the field as a time-of-day value. Pair with `renderAs: 'time'`. |
| `renderAs` | `'time'` | unset | Required for formatted time output. Without it, raw value renders as plain text. |
| `condition` | `(entity) => boolean` | unset | Hides the field for rows where the function returns `false`. |
| `sortable` | `boolean` | `false` | Enables column sorting. Sorts chronologically when `parseMode` is set. |
| `width` | CSS length or number | unset | Sets column width in table mode. |
| `align` | `'left'`, `'center'`, `'right'` | `'left'` | Text alignment in table mode. |

### `renderAsOptions` for `renderAs: 'time'`

| Option | Type | Default | Effect |
|---|---|---|---|
| `parseMode` | `'iso-time'` \| `'epoch-ms'` \| `'epoch-sec'` \| `'iso-datetime'` \| `'any'` | **required** | Declares the source format. Without this, renders `'Invalid'`. |
| `locale` | string | `'en-US'` | BCP 47 locale tag passed to `Intl.DateTimeFormat`. |
| `timezone` | `'local'` \| `'utc'` \| IANA string | `'local'` | Timezone for display. |
| `showSeconds` | `boolean` | `false` | Include seconds in the rendered time. |
| `hour12` | `boolean` | locale default | `true` for 12-hour AM/PM, `false` for 24-hour. |
| `formatOptions` | `Intl.DateTimeFormatOptions` | unset | Direct override — merged with computed options, takes precedence. |
| `fallbackText` | string | `'Invalid'` | Text shown when the value cannot be parsed. |

## Caveats

- `parseMode` is **required**. Without it, the field renders `'Invalid'`.
- `parseMode: 'iso-time'` expects `HH:mm:ss`. Values without seconds (`HH:mm`) may parse differently across environments — include seconds for reliability.
- `type: 'time'` has no `relativeTime` option — relative time is not meaningful for time-of-day values. Use `type: 'datetime'` or `type: 'timestamp'` for relative display.
- `timezone` affects display only. An `iso-time` value has no inherent timezone — the time is interpreted as-is in the requested timezone.
