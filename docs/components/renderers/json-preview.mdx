# jsonPreview Renderer

`renderAs: 'jsonPreview'` renders an object or array as an interactive collapsible JSON tree. String values that parse as JSON are automatically expanded.

## Core Examples

### Expanded by Default

<WidgemoShowcase
  description="Nested metadata object displayed as an expandable JSON tree."
  data={jsonData}
  config={jsonBasicConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{ key: 'meta', renderAs: 'jsonPreview', renderAsOptions: { defaultCollapsed: false } }
```

<div style={{ marginBottom: '1rem' }} />

### Collapsed with Depth Limit

`defaultCollapsed: true` folds nested nodes on initial render. `maxDepth` truncates structures deeper than the given level.

<WidgemoShowcase
  description="Same metadata, collapsed by default with a maxDepth of 2."
  data={jsonData}
  config={jsonCollapsedConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{ key: 'meta', renderAs: 'jsonPreview', renderAsOptions: { defaultCollapsed: true, maxDepth: 2 } }
```

## Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `defaultCollapsed` | boolean | `true` | Collapse nested nodes on initial render. The root object stays expanded. |
| `maxDepth` | number | `3` | Truncates nested structures beyond this depth. Must be `≥ 0`. |
| `className` | string | unset | Additional CSS class on the root element. |

## Caveats

- String values are attempted to be parsed as JSON. If parsing succeeds, the parsed structure is rendered as a tree. If it fails, the raw string is shown.
- Invalid JSON does not throw — the renderer falls back to a text representation.
- `maxDepth` uses the default of `3` if the value is negative (clamped to `0`).
- The JSON tree is read-only — values are not editable in-place.
