# link Renderer

`renderAs: 'link'` renders a field value as a navigable `<a>` element. Both `text` and `url` accept a function receiving the entity, enabling fully dynamic link construction.

## Core Examples

### Basic Link

<WidgemoShowcase
  description="Profile URL rendered as a fixed-text link opening in a new tab."
  data={linkData}
  config={linkBasicConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'profile',
  renderAs: 'link',
  renderAsOptions: { text: 'View profile', newTab: true },
}
```

<div style={{ marginBottom: '1rem' }} />

### Dynamic Link Text

`text` and `url` each accept `(entity) => string`. This lets you construct labels and URLs from any combination of row fields.

<WidgemoShowcase
  description="Link text is generated per-row using the entity name."
  data={linkData}
  config={linkDynamicConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'profile',
  renderAs: 'link',
  renderAsOptions: {
    text: (entity) => `Profile → ${entity.name}`,
    newTab: true,
  },
}
```

<div style={{ marginBottom: '1rem' }} />

### Custom URL (e.g. mailto)

`url` overrides the href independently of the field value. Use it to construct `mailto:`, `tel:`, or internal app routes from row data.

<WidgemoShowcase
  description="Link href is built from entity.email while the visible text is the email address."
  data={linkData}
  config={linkEmailConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'name',  // field value drives the display
  renderAs: 'link',
  renderAsOptions: {
    url: (entity) => `mailto:${entity.email}`,
    text: (entity) => entity.email,
  },
}
```

## Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `text` | string \| `(entity) => string` | field value | Visible link label. Falls back to the raw field value when omitted. |
| `url` | string \| `(entity) => string` | field value | href of the anchor. Falls back to the raw field value when omitted. |
| `newTab` | boolean \| `(entity) => boolean` | `false` | Opens the link in a new browser tab (`target="_blank"` + `rel="noopener noreferrer"`). |
| `externalWarning` | boolean | `false` | Shows a visual indicator when the URL navigates away from the current origin. |
| `className` | string | unset | Additional CSS class on the root element. |

## Caveats

- If both `text` and `url` are omitted, the raw field value is used as both the label and the href.
- `newTab: true` automatically adds `rel="noopener noreferrer"` to mitigate reverse tabnapping.
- Function forms of `text` and `url` receive the full entity object. Keep the functions pure — avoid side effects.
