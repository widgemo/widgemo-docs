# badge Renderer

`renderAs: 'badge'` renders a value as a styled pill or inline label. Use `colorMap` to map values to colors — any value not in the map falls back to `defaultColor`.

## Core Examples

### Basic Badge

<WidgemoShowcase
  description="Status column rendered as colored badges with a colorMap."
  data={badgeData}
  config={badgeBasicConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{
  key: 'status',
  renderAs: 'badge',
  renderAsOptions: {
    colorMap: {
      active:   { background: '#16a34a', text: '#ffffff' },
      pending:  { background: '#d97706', text: '#ffffff' },
      inactive: { background: '#6b7280', text: '#ffffff' },
    },
  },
}
```

<div style={{ marginBottom: '1rem' }} />

### Size Variants

`size` controls padding and font size. Default is `'md'`.

### Style Variants

`'badge'` renders a filled pill. `'inline'` renders colored text without a background chip.

  description="Badge style (filled pill) vs inline style (colored text)."

  config={badgeStyleConfig}
/>

<div style={{ marginBottom: '1rem' }} />

```tsx
{ renderAsOptions: { style: 'badge' } }   // filled pill
{ renderAsOptions: { style: 'inline' } }  // colored text only
```

## Options Reference

| Option | Type | Default | Effect |
|---|---|---|---|
| `colorMap` | `Record<string, string \| ColorConfig>` | `{}` | Maps field values to colors. String values are used as background color (text auto-contrasted). |
| `defaultColor` | `string \| ColorConfig` | `'#6c757d'` | Fallback color for values not in `colorMap`. |
| `style` | `'badge'` \| `'inline'` | `'badge'` | `'badge'`: filled pill. `'inline'`: colored text only. |
| `size` | `'sm'` \| `'md'` \| `'lg'` | `'md'` | Controls padding and font size. |
| `className` | string | unset | Additional CSS class on the root element. |

### `ColorConfig` shape

```ts
{
  background?: string;  // pill background color
  text?: string;        // label text color (auto-contrasted if omitted)
  icon?: string;        // icon name to prepend/append
  iconPosition?: 'left' | 'right' | 'only';  // default: 'left'
  iconColor?: string;   // icon color override
}
```

## Caveats

- An empty or nullish value renders a `-` placeholder — it does not throw.
- When `colorMap` maps a value to a plain string (e.g. `active: '#16a34a'`), text color is auto-contrasted against the background.
- `icon` within a `ColorConfig` expects an icon name registered with the Widgemo icon registry. Unrecognized names render nothing.
