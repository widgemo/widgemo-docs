# Extending Widgemo

This section covers how to extend Widgemo behavior without forking core.

## Extension Surfaces

- Custom renderers (`renderAs`)
- Custom field types
- Custom modes
- Custom icons
- Registry hooks and extension lifecycle behavior

## Recommended Order

1. Start with [Extension API](/core/reference/extension-api)
2. Implement one extension path:
- [Custom renderAs](/core/extensions/renderas-customization)
- [Custom Field Types](/core/extensions/custom-field-types)
- [Custom Modes](/core/guides/custom-modes)
- [Custom Icons](/core/guides/custom-icons)

## Built-in Assets

Before extending icons, review built-ins in [Built-in Icons](/core/components/icons/built-in-icons).
