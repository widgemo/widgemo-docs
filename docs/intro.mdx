# widgemo-core Documentation

Welcome to the **widgemo-core** documentation hub. This page explains how the docs site is organized so you can quickly find implementation details, configuration references, and extension guidance.

## How This Documentation Is Organized

- **Getting Started** — Start here to get aligned with package setup and the expected app integration flow.
  - `Installation` covers package setup, peer dependencies, and stylesheet import.
  - `Quick Start` provides the first working `Widgemo` setup.
  - `Integrate Into Existing App` focuses on adopting widgemo-core in an existing product.
- **Concepts** — Defines the core mental model used throughout the docs: modes, theming, DevMode, configuration model, and extension architecture.
- **Reference** — Source-of-truth API documentation for props and configuration (`widgemo-props`, `widgemo-config`, `content-config`, `field-config`, actions, CSS, and theme APIs).
- **Guides** — Task-oriented walkthroughs for real implementation scenarios (adoption workflow, temporal fields, composition bar).
- **Extensions & Customizations** — Advanced extension points for custom `renderAs`, custom field types, custom modes, custom icons, and extension API behavior.
- **Release Notes** — Migration and compatibility guidance for upgrades.

## Getting Started at a Glance

This intro is a fast orientation. The steps below are the absolute minimum to render Widgemo. For setup specifics and edge cases, use the Getting Started pages.

1. Install widgemo-core.
2. Import the stylesheet once at your app root.
3. Render `Widgemo` with `data` and a `zones`-based `config`.

```bash
npm install @widgemo/widgemo-core
```

```tsx
import { WidgemoThemeProvider, Widgemo } from '@widgemo/widgemo-core';
import '@widgemo/widgemo-core/style.css';

function App() {
  return (
    <WidgemoThemeProvider theme="light">
      <Widgemo data={data} config={config} />
    </WidgemoThemeProvider>
  );
}
```
